安装方法

方法一：命令行安装（推荐）

  1. 点击上方按钮下载 sonicnote-sync-1.0.0.vsix
  2. 打开终端，运行安装命令：
code --install-extension sonicnote-sync-1.0.0.vsix --force

如果 code 命令未配置，先在 VSCode 中按 ⇧⌘P，输入 Shell Command: Install 'code' command in PATH 进行安装。

方法二：界面安装

  1. 下载 .vsix 文件
  2. 打开 VSCode → 扩展面板 (⇧⌘X) → 点击右上角 ⋯ → 从 VSIX 安装...
  3. 选择下载的 .vsix 文件完成安装
提示：安装前请关闭 VSCode 中已打开的 SonicNoteSync 插件实例（如有），避免冲突。

快速开始

  1. 点击 VSCode 左侧活动栏的 SonicNote Sync 图标打开侧边栏
  2. 在设置面板中填入 API 地址 和 MCP Key（获取方式见下方）
  3. 点击 登录 完成认证
  4. 点击 选择同步文件夹 设置 Markdown 文件存放目录
  5. 点击 同步录音 拉取录音并生成 Markdown 文件

MCP Key 获取方式

  1. 打开妙记 App
  2. 进入 我的 → MCP Key 管理
  3. 点击 创建 Key，复制生成的 Key（格式：sk-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx）
  4. 回到 VSCode 插件面板，粘贴 Key 后点击登录

常见问题

Q: 登录失败提示 "认证失败"？

A: 请检查 MCP Key 是否正确，以及 API 地址是否可访问。Key 格式应为 sk- 开头。

Q: 同步后没有生成文件？

A: 请检查同步文件夹路径是否有写入权限，以及录音列表中是否有已转录完成的录音。

Q: 插件侧边栏显示空白？

A: 请确认 VSCode 版本 >= 1.85.0，WebviewView 需要此版本以上支持。也可尝试重新加载窗口 (⌘R)。

Q: 如何更新插件？

A: 下载最新版 .vsix 文件，用相同命令重新安装即可覆盖旧版本。

更新方法

下载最新版安装包，运行 code --install-extension sonicnote-sync-<版本号>.vsix --force 即可覆盖安装，无需先卸载旧版。
